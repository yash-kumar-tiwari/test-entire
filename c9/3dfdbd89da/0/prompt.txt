# Privacy Requirements (Very Important)

Treat security as a real requirement.

Users must never be able to:

* Read another user's bookmarks
* Edit another user's bookmarks
* Delete another user's bookmarks

Assume users can:

* Change URLs
* Modify requests
* Call APIs directly

Security must be enforced at:

1. Database level
2. API level
3. UI level

Use Supabase Row Level Security (RLS).

Implement proper policies.

A user should only access records where:

bookmarks.user_id = auth.uid()

Do not rely only on frontend checks.
